# AGENTS.md

**Potato Head** — Rust workspace + PyO3/Maturin Python bindings providing a multi-provider LLM client (OpenAI, Gemini, Vertex AI, Anthropic), agent orchestration, and workflow management. Core utility for [opsml](https://github.com/demml/opsml) and [scouter](https://github.com/demml/scouter).

## Setup

```bash
# Rust (repo root)
cargo build --workspace

# Python extension (from py-potato/)
make setup.project   # uv sync + maturin develop --release
```

After any Rust change affecting the Python extension, re-run `make setup.project` in `py-potato/`.

## Architecture

### Crate Dependency Flow

```
potato-type → potato-provider → potato-agent → potato-workflow → potato-head (facade)
```

Supporting crates:
- **potato-state** — `PotatoState` singleton holding one Tokio runtime (`OnceLock<Runtime>`); bridges async Rust ↔ Python sync via `block_on()`
- **potato-util** — UUID v7, JSON helpers, weighted scoring, `PyHelperFuncs`
- **potatohead-macro** — proc macros `try_extract_py_object!` and `extract_and_push!` for PyO3 enum extraction
- **baked-potato** — mock servers + integration tests; NOT published to crates.io

### Python Binding Layer (`py-potato/`)

- `py-potato/src/lib.rs` — PyO3 module root; registers all Rust types and submodules
- `py-potato/python/potato_head/__init__.py` — re-exports the Rust extension to Python
- `py-potato/python/potato_head/_potato_head.pyi` — auto-generated type stubs (do not hand-edit)
- Provider subpackages: `openai/`, `google/`, `anthropic/` under `potato_head/`
- Extension module name: `_potato_head` (ABI3, Python ≥ 3.10)

### Provider Abstraction

`GenAiClient` (`potato-provider`) is a Rust enum: `OpenAI | Gemini | Vertex | Anthropic`. Each variant wraps a provider-specific HTTP client handling auth, serialization, and streaming. `ChatResponse` is a parallel enum unifying provider response types.

### Orchestration Patterns

- **SequentialAgent** — runs agents serially; each output feeds the next
- **ParallelAgent** — runs agents concurrently with a configurable `MergeStrategy`
- **Workflow** — DAG execution; `Task.dependencies` holds upstream task IDs; tasks self-track `TaskStatus` (Pending → Running → Completed/Failed)
- **Callbacks** (`AgentCallback` trait) — fire at lifecycle points (started, completed, failed)
- **Completion criteria** (`CompletionCriteria` trait) — keyword match, structured output, max iterations

### Memory System

Implementations of the `Memory` trait (`Send + Sync`):
- `InMemoryMemory` — unbounded history
- `WindowedMemory` — sliding window over last N turns
- `PersistentMemory` — backed by `SessionStore` (SQLite via `feature = "sqlite"`)

Memory lives behind `Arc<tokio::sync::Mutex<Box<dyn Memory>>>` on the Agent.

## Critical Constraints

**Async/sync bridge:** All PyO3 `#[pymethods]` must call `potato_state::block_on(future)`. Never create an ad-hoc Tokio runtime.

**Python extension builds:** Never use `cargo build` to build the Python extension. Always use `maturin develop` (via `make setup.project`).

**Type stubs:** `_potato_head.pyi` is auto-generated by `scripts/create_stubs.py` — do not hand-edit. Regenerate after API changes.

**Test isolation:** Always run Rust tests with `--test-threads=1` (especially store/SQLite tests).

**Publishing:** All crates except `baked-potato` publish to crates.io via `release-plz`. Do not publish manually.

## Adding a New Provider

1. Add variant to `Provider` enum — `crates/potato_type/src/lib.rs`
2. Add variant to `GenAiClient` — `crates/potato_provider/src/providers/client.rs`
3. Implement `crates/potato_provider/src/providers/<name>/client.rs` (auth, HTTP, streaming)
4. Add provider message types in `crates/potato_type/src/providers/<name>/`
5. Register PyO3 submodule in `py-potato/src/lib.rs`
6. Add Python subpackage under `py-potato/python/potato_head/<name>/`

## Code Style

- **Rust:** `cargo fmt` + `clippy -D warnings` must pass. Use `tracing` for logging, not `println!`. Use `thiserror` for error enums; implement `From<XxxError> for PyErr` for all types exposed to Python.
- **Python:** `black` (line-length 120), `isort` (black profile), `ruff`, `mypy`. Full type coverage required.
- No comments explaining what obviously-readable code does.
